# SVG Logo 生成器

用代码写 Logo。不画图。

画图是概率。写代码是确定性。SVG 精确可控、可编辑、矢量无损。

## 设计原则（指导方向，不是死规则）

1. **简洁优先** — 好 Logo 不一定元素少，但每个元素都有存在的理由
2. **留白是设计的一部分** — 不要把画布填满。留白让 Logo 呼吸
3. **不放文字** — SVG 里不写任何文字。文字后期用设计工具加
4. **考虑缩小** — Logo 缩小到 32px 还能辨认才算合格

## 红线（这些是真正不能碰的）

1. **禁止在 SVG 里放文字** — AI 写的 SVG 文字依赖字体，换环境必变形
2. **禁止超过 10 个独立元素** — 太多就不是 Logo 了，是插画
3. **禁止用渐变模拟 3D 效果** — Logo 应该是扁平的，3D 感用 Figma 后期加

## 执行

### 阶段一：信息收集

问用户以下信息（如果用户已经提供了部分，跳过已有的）：

1. **产品名称**
2. **一句话描述**（做什么的）
3. **目标受众**（给谁用的）
4. **核心概念**（1-2 个关键词，如"连接""流动""安全""简洁"）
5. **设计偏好**（可选）：
   - 风格：极简 / 几何 / 有机 / 科技
   - 色温：冷色 / 暖色 / 中性
   - 线条：纤细 / 中等 / 粗重

如果用户没给设计偏好，根据产品类型推断合理的默认值。

### 阶段二：生成 6 个 SVG 变体

基于收集的信息，参考 `references/design-patterns.md` 中的设计模式库，生成至少 6 个不同风格的 SVG Logo。

**设计模式选择：** 不要每次都用固定的 6 种。根据产品特点，从模式库中选择最合适的 3-4 种模式，每种模式生成 1-2 个变体。也可以混合多种模式。

**SVG 规范：**
- viewBox 统一用 `0 0 100 100`（和 references 中的示例一致）
- 颜色用 `currentColor`（让 Logo 自适应父容器颜色）
- 线条末端（round/square/butt）根据品牌个性选择
- 先用纯黑设计，形状满意后再加颜色

**每个变体必须包含：**
- 完整的 SVG 代码（`viewBox="0 0 100 100"`）
- 一句话设计理念（为什么这样设计）

**生成完 6 个变体后，必须创建一个 HTML 预览页面（这一步不能跳过）：**
- 先读取 `assets/variants-template.html` 模板文件
- 将模板中的 `<!-- SVG_1 -->` 到 `<!-- SVG_6 -->` 替换为实际 SVG 代码
- 将 `<!-- DESC_1 -->` 到 `<!-- DESC_6 -->` 替换为一句话设计理念
- 将 `<!-- PRODUCT_NAME -->` 替换为产品名称
- 页面包含 hover 效果和流畅过渡动画
- 保存到 `~/Downloads/logo-variants.html`
- **写完文件后，告诉用户：请在浏览器中打开 ~/Downloads/logo-variants.html 对比 6 个方案**

告诉用户：打开这个 HTML 文件对比 6 个方案，选一个最喜欢的。

### 阶段三：迭代优化

用户选定一个方案后，生成 3 个微调变体：

- 变体 A：调整尺寸/间距（更紧凑或更松散）
- 变体 B：调整颜色（用户指定或推荐替代色）
- 变体 C：调整细节（线条粗细、圆角、末端处理）

更新 HTML 预览页，展示 3 个微调变体。

用户可以继续迭代。每轮只改 1-2 个参数。

### 阶段四：黑白测试 + 交付

用户确认最终方案后：

1. **生成 3 个版本的 SVG 文件：**
   - `logo.svg` — 原始配色版
   - `logo-black.svg` — 纯黑版（所有颜色改为 #111111）
   - `logo-white.svg` — 纯白版（所有颜色改为 #ffffff，加黑色背景矩形）

2. **生成最终预览 HTML（不能跳过）：**
   - 三个版本并排展示（原色 / 纯黑白底 / 纯白黑底）
   - 保存到 `~/Downloads/logo-final.html`
   - 告诉用户在浏览器中打开查看

3. **黑白测试判断：**
   - 如果纯黑版在白底上辨识度不够 → 告诉用户"设计可能还不够简洁，建议继续简化"
   - 如果通过 → 告诉用户"黑白测试通过，Logo 足够简洁"

4. **所有 SVG 文件保存到 `~/Downloads/`**

5. **输出交付清单：**
   ```
   ✅ Logo 交付完成
   
   📁 文件列表：
   - ~/Downloads/logo.svg（原始配色，可编辑矢量）
   - ~/Downloads/logo-black.svg（纯黑版）
   - ~/Downloads/logo-white.svg（纯白版）
   - ~/Downloads/logo-final.html（三版对比预览）
   
   📋 下一步建议：
   - 复制 SVG 代码粘贴到 Figma，可以加渐变、阴影等精修
   - 用 AI 绘图工具生成展示图（配专业背景）
   - 用设计工具加上品牌名文字
   ```

### 阶段五（可选）：展示图

如果用户想要展示图，参考 `references/background-styles.md` 中的 12 种背景风格。

1. 根据产品类型推荐 2-4 种背景风格
2. 为每种风格生成 AI 绘图提示词（英文），用户可以粘贴到 ChatGPT / Gemini 生成展示图
3. 提示词格式：`Professional logo showcase: [Logo description] centered on [background style description]. Studio lighting, 4K, photorealistic background with flat vector logo overlay.`

## 模板

- 生成 6 变体的 HTML 预览页时，参考 `assets/variants-template.html` 的布局结构
- 将 `<!-- SVG_N -->` 替换为实际 SVG 代码，`<!-- DESC_N -->` 替换为设计理念，`<!-- PRODUCT_NAME -->` 替换为产品名

## 验收

- **简洁**：元素不超过 2 个
- **可辨**：缩小到 32x32 还能认出来
- **黑白通过**：去掉颜色也能辨认
- **可编辑**：SVG 代码结构清晰，粘贴到 Figma 后每个元素独立可选
